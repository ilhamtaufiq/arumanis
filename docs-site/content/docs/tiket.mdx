## Apa itu tiket?

Tiket mencatat **kendala, temuan, atau permintaan tindak lanjut** terkait paket pekerjaan (misalnya keterlambatan, isu teknis, butuh klarifikasi dokumen).

## Daftar tiket

1. Buka **Tiket** di sidebar
2. Gunakan filter status / pekerjaan / rentang tanggal jika tersedia
3. Klik baris untuk membuka detail + riwayat komentar

## Membuat tiket

1. Tombol **Buat tiket** / **Tambah**
2. Isi minimal:
   - **Judul** — singkat dan jelas
   - **Pekerjaan** — paket terkait
   - **Deskripsi** — kronologi / langkah yang sudah dicoba
   - **Prioritas** (jika ada)
3. Lampirkan foto/file jika diminta form
4. Simpan

## Menindaklanjuti

| Aksi | Keterangan |
|------|------------|
| Komentar | Update progress diskusi |
| Ubah status | Open → proses → selesai / ditutup |
| Assign | Tugaskan ke user/role (jika fitur aktif) |

## Dari Panel Pengawasan

Pengawas dapat membuat/melihat tiket di menu **Tiket** Panel Pengawasan dan tab Tiket pada detail pekerjaan. Lihat [Panel Pengawasan](/docs/pengawas-panel).

## Tips

- Satu isu = satu tiket (jangan campur banyak topik)
- Sebut **kode/nama paket** di judul bila memungkinkan
- Tutup tiket hanya setelah ada resolusi yang disepakati
