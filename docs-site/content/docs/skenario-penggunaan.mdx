---
title: Skenario Penggunaan
description: Alur kerja tipikal per peran (admin, operator, pengawas).
---

## Admin sistem

1. Login → [Dashboard](/docs/dashboard)
2. Kelola [Users](/docs/users) & [Roles](/docs/roles-permissions)
3. Atur [Settings](/docs/settings) (tahun, backup)
4. Pantau [Audit logs](/docs/settings#audit--error)
5. Opsional: hubungkan [WhatsApp](/docs/whatsapp), cek [OnlyOffice](/docs/dokumen-onlyoffice)

## Operator / PPTK

1. Input [Kegiatan](/docs/kegiatan)
2. Buat/update [Pekerjaan](/docs/pekerjaan-output) + output
3. Isi [Kontrak](/docs/kontrak) & [Penyedia](/docs/penerima-penyedia)
4. Unggah [Berkas & Foto](/docs/berkas-foto); preview lewat [OnlyOffice](/docs/dokumen-onlyoffice)
5. Cek pagu/sub kegiatan di [SIPD Renja](/docs/sipd-renja); sync kegiatan bila perlu
6. Import dokumen pengadaan dari [SPSE](/docs/spse-import) jika paket non-tender
7. Lengkapi [Checklist](/docs/checklist)
8. Tangani [Tiket](/docs/tiket) dari lapangan

## Pengawas lapangan

1. Login Arumanis → SSO ke [Panel Pengawasan](/docs/pengawas-panel)
2. Pilih paket → isi progress & foto slot
3. Buat laporan mingguan
4. Buat tiket kendala bila perlu
5. Logout / biarkan sesi cookie (jangan bagikan perangkat)

## Viewer / eksekutif

1. Login dengan role terbatas
2. Baca dashboard / rekap / [SPM](/docs/spm-capaian)
3. Unduh laporan yang diizinkan (tanpa edit)

## Evaluasi Puspen

1. Masuk [Puspen](/docs/puspen)
2. Review progress fisik & KPI pengawas
3. Bagikan media / tanda tangani PDF bila alur meminta
